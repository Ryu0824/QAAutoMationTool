# QAAutoMationTool

게임 데일리 루틴(숙제) 자동화를 위한 설치형 데스크탑 툴.
코딩 없이 **비주얼 노드 에디터**로 자동화 플로우를 구성하고 저장·실행할 수 있습니다.

---

## 주요 기능

- **비주얼 노드 에디터** — 노드를 드래그&드롭으로 배치하고 연결해 플로우 구성
- **블랙박스 테스트 방식** — 화면 캡처 기반 이미지 인식 및 OCR로 게임 상태 판단
- **플로우 영구 저장** — 한 번 설정한 루틴은 로컬에 저장되어 재사용 가능
- **플로우 이름 변경** — 플로우 목록에서 인라인으로 이름 편집 가능
- **미저장 변경 보호** — 플로우 전환·종료 시 미저장 변경 사항이 있으면 확인 팝업 표시
- **인앱 화면 캡처 도구** — 투명 오버레이로 게임 화면에서 좌표·영역을 직접 선택해 노드에 적용
- **PC 게임 우선 지원**, 이후 모바일(ADB) 확장 예정

## 노드 타입

| 구분 | 노드 | 설명 |
|---|---|---|
| **Action** | Click | 마우스 단일/더블/우클릭 |
| | KeyInput | 키보드 입력 (단일키, 조합키) |
| | Drag | 드래그 동작 |
| | Delay | 지정 시간 대기 |
| | FocusWindow | 대상 프로그램 창 포커스 |
| **Condition** | ImageMatch | 화면에서 템플릿 이미지 탐색 → 분기 |
| | WaitForImage | 이미지가 나타날 때까지 폴링 대기 → 발견/타임아웃 분기 |
| | OCRRead | 지정 영역 텍스트 읽기 → 분기 |
| **Control** | Start | 플로우 시작점 |
| | Loop | 횟수 또는 조건 충족까지 반복 |
| | Sequence | 순차 실행 |
| | SubFlow | 다른 저장된 플로우를 서브루틴으로 호출 |

## 시작하기

### 요구사항

- Node.js 18+
- Python 3.8+ (`opencv-python`, `numpy` 필요)
- Windows 10/11

### 설치

```bash
# Node 의존성 설치
npm install

# Python 의존성 설치
pip install -r python/requirements.txt
```

### 실행

```bash
# 개발 모드
npm run dev

# 프로덕션 빌드
npm run build

# 설치 파일 패키징
npm run dist
```

> **관리자 권한 실행 권장**
> 게임 런처 등 관리자 권한 프로세스와 상호작용하려면 터미널을 관리자 권한으로 실행하세요.

## 사용 방법

1. 왼쪽 패널에서 노드를 드래그해 캔버스에 배치
2. 노드 클릭 → 오른쪽 속성 패널에서 값 설정
3. 노드 간 핸들을 연결해 플로우 구성
4. **저장** 버튼으로 플로우 저장
5. **▶ 실행** 버튼으로 플로우 실행
6. 하단 로그 패널에서 실행 과정 확인

### 화면에서 직접 선택

속성 패널의 **"화면에서 선택"** 버튼을 누르면 투명 오버레이가 열립니다.

- **Click / Drag 노드** — 게임 화면에서 클릭해 좌표를 자동 입력
- **ImageMatch / WaitForImage 노드** — 영역을 드래그해 템플릿 이미지를 캡처 후 저장

### 변수 전달

`ImageMatch` 노드에서 발견한 좌표를 변수에 저장하고, `Click` / `Drag` 노드에서 해당 변수를 참조할 수 있습니다.

```
ImageMatch (varName: "target") → found → Click (변수 사용: "target")
```
